HTML

  • Start a webpage with the declaration <!DOCTYPE html> to let the browser know what to expect for this webpage
  • The head element contains information about the webpage
  • The header element is not to be confused with the head element. The header element is part of the body element, and its purpose is to identify the content that precedes the primary content (the main and footer elements) of the webpage
  • The footer element is at the bottom of the body element, and usually contains info such as copyright, author, contact, sitemap, and navigation
  • The title element lives inside the head element, and defines the title for a webpage, which is important for SEO
  • The body element represents the visible content shown to the user
  • The style element defines the CSS (Cascading Style Sheet) styles associated with a website
  • The link element connects an external resource (including text/css files) to the HTML document
  • The metadata of a webpage is always located inside the head element, in the meta element
  • Common attributes of the meta element include: character set, description, keywords, author, and viewport
  • You can insert comments for other website developers by enclosing the message in
    <! -- YOUR COMMENT -->, comments will only be visible in source code and will not be rendered by the browser
  • The <br> element is an empty element that creates a line break
  • Every html element can have an id attribute
  • The <a> element creates a link (anchor) to the same page or other pages, and must include the actual URL link as an attribute. For example: <a href="URL">.
    Links to certain sections on same page can be expressed as <a href=#id>
  • The <img> element is for images, and must include the attributes of: src (source), alt (alt text), width, and height (in pixels)

CSS

  • Inline CSS info can be anywhere in the html file by using the style attribute with a CSS rule.
    For example: <h1 style= "color:red;">h1</h1>
  • Internal and external CSS info sit inside the head of the HTML page
  • External CSS files must be linked to the html page in the head element.
    For example: <link rel="stylesheet" href="./asset/style.css" type="text/css">
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Padding sits around the element, and border sits around the padding, and margin sits around border
  • Increasing the padding of an element effectively increases the entire element
  • Using margin-left: auto and margin-right: auto together centeres the element
  • Padding-left can be used to control indentation (for <ul> or <ol>, for example)
  • Font-size can be used since default font size is 16px
  • Use the asterisk symbol * as universal selector targeting all the elements
  • Use commas to separate different elements you want to select in the selector. For example: header, footer or h1, h2
  • Use the period symbol . followed by class name to select a elements that carry the same class attribute
  • Color indicates the text color of the element, while background-color indicates the backdrop color of the element
  • For img selector, display: block will make sure the image starts on a new line
  • The text-align property can be used to specify text alignment (center, right, left, justify)
  • Border thickness and color can be controlled in pixels and color name or code. For example: {border: 5px solid grey;}
  • Box-shadow property customizes outside of the border and the value format is: X px Y px #color_code.
    For example: {box-shadow: 5px 10px #888888;} creates a light grey box shadow that's 5px horizontally and 10px vertically

Git

Common Git Commands:

  • git status: checks which branch we are currently on
  • git checkout -b "branch-name": creates a new branch and switches to it
  • git add -A: stages all changes for the branch for commits
  • git pull origin main: checks to see if your local branch is up to date with the origin's main branch
  • git commit -m"message": confirms the changes to be pushed onto Github with a message that describes the changes
  • git push origin "new-branch": syncs the new branch with Github
  • git branch -d branch_name deletes a branch. Git will not let you delete the branch you're currently checking out, so make sure you check out another branch (for example, main) before running this git command
  • git branch -d branch_name will not let you delete a branch until it's commited and pushed. If you need to delete a branch that hasn't been commited and pushed, run this git command: git branch -D branch_name

JavaScript

  • link Javascript files at the bottom of the body element with the script element like so: <script src = "file location"></script>
  • Variables are containers that store values, and you can name your variable almost anything. Use semicolons to separate statements
  • JavaScripts are run by the computer from top to bottom
  • Declare a variable by typing var, followed by the variable name. To assign values to variables, follow with “=” + value.
    For example: var MyVarialbe = 'Bob'. Retrive values by calling variable names.
  • Code blocks are placed in curly brackets
  • JavaScript is case-sensitive
  • End each line in JacaScript with a semicolon
  • Different types of JavaScript data:
    1. String: a sequence of text. For example: let MyVariable = 'Bob'
    2. Number: numbers do not need quote marks
    3. Boolean: true or false. Also doesn't need quote marks
    4. Array: this allows us to store multiple values in a single reference.
      For example: let MyVariable = [1, 'bob', 'steve', 10]. When calling a variable value in an array, use the index number, which is the position of the data in the array, with the first item being index [0]. For example: MyVariable[0], MyVariablew[1], MyVariable[2], etc. The index number of an array is not to be confused with the length of the array. The length of the array is the number of items in the array, while the first item in an array has the index number of 0.
    5. Object: this can be anything. Everything in JavaScript is an object and can be stored in a variable. For example: let MyVariable = document.querySelector('h1')
  • There are two categories of JavaScript values
      Primitive Values: immutable datum
    1. Boolean Type: true or false
    2. Null Type: null, intentional absence of value
    3. Undefined Type: any variable that hasn't been assigned a value
    4. Number Type
    5. BitInt Type
    6. String Type: textual data
    7. Symbol Type
      Objects
  • An object is a value in memory that is referenced by an identifier
  • Insert comments in one of these two ways: /* YOUR COMMENT */ or // YOUR COMMENT
  • Operators are math symbols that produce results based on two values:
    1. Addition: +
    2. Subtraction, Multiplication, Division: -, *, /
    3. Assignment: =
    4. Strict Equality - True or False (Boolean results): ===
    5. Not, Does Not Equal: ! or !==. Used to test if two values are not equal
  • Conditionals are code structures used to test if an expression returns true or false
  • Functions are packaged functionality that you wish to reuse. For example: a body of codes that are run when you call on the function name, and they are followed by parentheses
  • Events: real interactivity on a website requires event handlers: code structures that listen for activity in the brower and run codes in response. For example: the click event
  • Variants can be assigned and re-assigned any value and any type of value

Common JavaScript Methods and Functions:

  • Console.log(variable name): a method used to output messages to browser consoles and to test codes. Note that the variable names we put inside the parentheses are called Arguments
  • Conditional: if (var = X) { run code A; } else if (var = Y) { run code B; } else { run code C; }
  • "++" is equal to "+1". For example: X++ means X+1
  • The built-in property "length" allows you to run for loops that depends on the length of your array. For example: variableName.length
  • "For loops" start with "for", follow by a three-part conditional sentences enclosed in parentheses, and then by a code block enclosed in curly brackets. For example: for(varName X = 0; X < varName.length; X++) { Code Block } This will loop the code multiple times, with X incrementing by 1 after each loop, as long as X is smaller than the length of the variant

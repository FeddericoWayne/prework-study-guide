Prework Study Guide

HTML

  • Start a webpage with the declaration <!DOCTYPE html> to let the browser know what to expect for this webpage
  • The head element contains information about the webpage
  • The header element is not to be confused with the head element. The header element is part of the body element, and its purpose is to identify the content that precedes the primary content (the main and footer elements) of the webpage
  • The footer element is at the bottom of the body element, and usually contains info such as copyright, author, contact, sitemap, and navigation
  • The title element lives inside the head element, and defines the title for a webpage, which is important for SEO
  • The body element represents the visible content shown to the user
  • The style element defines the CSS (Cascading Style Sheet) styles associated with a website
  • The link element connects an external resource (including text/css files) to the HTML document
  • The metadata of a webpage is always located inside the head element, in the meta element
  • Common attributes of the meta element include: character set, description, keywords, author, and viewport
  • You can insert comments for other website developers by enclosing the message in
    <! -- YOUR COMMENT -->, comments will only be visible in source code and will not be rendered by the browser
  • The <br> element is an empty element that creates a line break
  • Every html element can have an id attribute
  • The <a> element creates a link (anchor) to the same page or other pages, and must include the actual URL link as an attribute. For example: <a href="URL">.
    Links to certain sections on same page can be expressed as <a href=#id>
  • The <img> element is for images, and must include the attributes of: src (source), alt (alt text), width, and height (in pixels)

CSS

  • Inline CSS info can be anywhere in the html file by using the style attribute with a CSS rule.
    For example: <h1 style= "color:red;">h1</h1>
  • Internal and external CSS info sit inside the head of the HTML page
  • External CSS files must be linked to the html page in the head element.
    For example: <link rel="stylesheet" href="./asset/style.css" type="text/css">
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • padding sits around the element, and border sits around the padding, and margin sits around border
  • Increasing the padding of an element effectively increases the entire element
  • Using margin-left: auto and margin-right: auto together centeres the element
  • padding-left can be used to control indentation (for <ul> or <ol>, for example)
  • font-size can be used since default font size is 16px
  • Use the asterisk symbol * as universal selector targeting all the elements
  • Use commas to separate different elements you want to select in the selector. For example: header, footer or h1, h2
  • Use the period symbol . followed by class name to select a elements that carry the same class attribute
  • Color indicates the text color of the element, while background-color indicates the backdrop color of the element
  • For img selector, display: block will make sure the image starts on a new line
  • The text-align property can be used to specify text alignment (center, right, left, justify)
  • border thickness and color can be controlled in pixels and color name or code. For example: {border: 5px solid grey;}
  • box-shadow property customizes outside of the border and the value format is: X px Y px #color_code.
    For example: {box-shadow: 5px 10px #888888;} creates a light grey box shadow that's 5px horizontally and 10px vertically

Git

Common Git Commands:

  • git status: checks which branch we are currently on
  • git checkout -b "branch-name": creates a new branch and switches to it
  • git add -A: stages all changes for the branch for commits
  • git pull origin main: checks to see if your local branch is up to date with the origin's main branch
  • git commit -m"message": confirms the changes to be pushed onto Github with a message that describes the changes
  • git push origin "new-branch": syncs the new branch with Github
  • git branch -d branch_name deletes a branch
